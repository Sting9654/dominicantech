Como la arquitectura de mi proyecto me complica un poco la entrega en PDF y la correcta ejecución se lo voy a envíar en Archivo rar para que pueda probarlo.

Las direcciones mi router las maneja distinto ejemplo:

http://localhost/Diplomado%20Back-end/dominicanatech/index.php?controller=Product&action=index

Donde el controller este parametro solo llevara el primer nombre si es ClientController solo será Client su valor y en el action si tiene solamente se pondra si el index o cualquier otro que ya este previamente declarado.

En caso de Preguntas con respecto a las conexiones con base de datos se realizan por medio de un fichero que sea de nombre config.env pues hay una función para extraer datos de ahí.

un ejemplo de como va la organización de los parametros a buscar sería

DB_HOST=host donde se encuentra alojada la base de datos
DB_USERNAME=el nombre de usuario
DB_PASSWORD=contraseña si tiene
DB_NAME=nombre de la base de datos

APP_DEBUG=true
APP_URL=http://localhost

En caso de cualquier retroalimentacion quedo al pendiente!
